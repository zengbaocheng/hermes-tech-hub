# gbrain 模式技能

> 出自 `garrytan/gbrain` — Hermes Agent & OpenClaw 的 Brain 系统

## 核心架构

gbrain 的 Brain 由三部分组成：

1. **技能层** — 43 个 Hermes 原生技能，按功能分组
2. **数据层** — 本地 SQLite 数据库 + 向量索引
3. **连接层** — 实体提取器自动从会话中建立知识图谱

## 可复用的关键模式

### 模式 1：自连接知识图谱

实体提取器（Entity Extractor）是 gbrain 的核心创新：

```
对话 → 实体提取器 → 实体（人员/项目/概念/代码）→ 关系边（引用/依赖/隶属）
                    ↓
              图谱查询 → 回答问题时提供上下文
```

**实施方法**：
1. 每次会话中提取名词实体（项目名、API、类、术语）
2. 建立实体间的关系边（X 调用 Y、X 依赖于 Z）
3. 新会话时查询相关实体及其连接

### 模式 2：混合搜索（三重检索）

```
用户问题
    ↓
┌─────────────────────────┐
│  ❶ 关键词搜索（FTS5）    │
│  ❷ 向量搜索（嵌入）      │
│  ❸ 图谱搜索（关系边）    │
└────────┬────────────────┘
         ↓
   结果融合 + 重排序
         ↓
      最终上下文
```

### 模式 3：Troves 收藏系统

类似浏览器的「收藏夹」，但针对 Agent 查询结果：

- 可收藏查询结果、代码片段、知识发现
- 按标签组织（`tag:module`, `tag:architecture`）
- 跨会话持久化

### 模式 4：记忆诊断

- 检查知识图谱中有多少孤立的实体（没有边的节点）
- 检查记忆覆盖率（哪些主题缺乏记录）
- 主动建议需要深入记录的主题

### 模式 5：Planner 规划器

- 将大任务分解为记忆感知的步骤
- 每一步检查已有记忆能否加速当前步骤
- 完成后自动总结并存入记忆

## 配置

gbrain 的配置在 `~/.gbrain/config.yaml`，数据在 `~/.gbrain/data/`

## 注意事项

- 43 个技能全部是 Hermes 原生 SKILL.md 格式，可以逐技能复制使用
- 自连接图谱的实体提取依赖于高质量的对话记录
- Troves 需要主动管理，否则会劣化为无标签的大杂烩
