# 系统依赖文件自动备份机制

## 功能概述

1. **定期备份**：每周日2点自动备份系统依赖文件
2. **只备份系统文件**：不备份任务生成文件
3. **保留2周**：只保留最近2个周日的备份
4. **自动清理**：第3周自动删除第1周的备份，无需授权

## 备份范围

### 必须备份：系统依赖文件

| 路径 | 说明 |
|------|------|
| ~/.hermes/config.yaml | Hermes 主配置文件 |
| ~/.hermes/.env | API 密钥配置 |
| ~/.hermes/auth.json | 认证文件 |
| ~/.hermes/vendor_models.json | 模型状态文件 |
| ~/.hermes/skills/ | 所有技能文件 |
| ~/.hermes/hermes-agent/ | Hermes 源代码 |
| ~/.hermes/cron/ | 定时任务配置 |
| ~/.hermes/profiles/ | Profile 配置 |
| /home/zbc/.ssh/ | SSH 密钥 |

### 不备份：任务生成文件

- /home/zbc/*.docx, *.pdf, *.txt 等文档
- /home/zbc/downloads/
- /home/zbc/images/
- ~/.hermes/sessions/
- ~/.hermes/logs/

## 备份存储位置

```
~/.hermes/backups/
├── 2026-04-27_02-00/          # 第1周备份
│   ├── config.yaml
│   ├── .env (加密)
│   ├── skills/
│   └── ...
├── 2026-05-04_02-00/          # 第2周备份
│   ├── config.yaml
│   └── ...
└── 2026-05-11_02-00/          # 第3周备份（将自动删除第1周）
    └── ...
```

## 执行流程

```
每周日2点触发
         │
         ▼
┌─────────────────────────┐
│ 创建新备份目录          │
│ (YYYY-MM-DD_HH-MM)     │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 备份系统依赖文件        │
│ (逐个复制到备份目录)    │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 统计备份数量            │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 备份数量 > 2?           │
│ 是 → 删除最早的备份     │
│ 否 → 结束               │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 向主上汇报结果          │
└─────────────────────────┘
```

## 备份规则

1. **每周备份**：每周日2点自动执行
2. **保留2周**：只保留最近2个周日的备份
3. **自动清理**：第3周自动删除第1周备份，无需授权
4. **加密敏感文件**：.env 文件可以加密备份（可选）

## 清理规则

- 备份目录数量 > 2 时，自动删除最早的备份目录
- 不需要主上授权，自动执行
- 删除后向主上汇报已删除的备份

## 手动备份

加载 skill 后可以直接说：
- "执行系统备份"
- "备份系统文件"

## 恢复备份

如需恢复备份，执行：
```bash
# 查看可用备份
ls -la ~/.hermes/backups/

# 恢复配置（示例）
cp ~/.hermes/backups/2026-05-04_02-00/config.yaml ~/.hermes/config.yaml
```

## 注意事项

1. .env 文件包含敏感 API Key，建议加密存储
2. skills 目录备份可能较大，首次备份请确认空间充足
3. 备份自动清理是单向的，删除后无法恢复

## 已知陷阱

### 1. hermes-agent 目录超大，cp -r 会超时
hermes-agent/ 包含 12,000+ 文件，约 443MB。使用 `cp -r` 会在 120s 超时。

**解决方法**：使用 rsync 代替 cp -r，或者分批复制：
```bash
rsync -a ~/.hermes/hermes-agent/ "$BACKUP_DIR/hermes-agent/"
```
如果 rsync 也不可用，可以先复制关键子目录，再单独 rsync hermes-agent。

### 2. profiles/ 目录可能不存在
`~/.hermes/profiles/` 在某些安装中不存在。备份时应先检查存在性：
```bash
[ -d ~/.hermes/profiles ] && cp -r ~/.hermes/profiles "$BACKUP_DIR/"
```

### 3. 备份目录创建失败
如果 `~/.hermes/backups/YYYY-MM-DD_HH-MM/` 已被占用（可能是之前失败的残留），mkdir 会失败。
**解决方法**：先删除残留文件或使用 `mkdir -p` 确保是目录：
```bash
rm -rf ~/.hermes/backups/YYYY-MM-DD_HH-MM  # 删除可能的残留文件
mkdir -p ~/.hermes/backups/YYYY-MM-DD_HH-MM
```

### 4. 清理逻辑只针对备份目录数量，而非时间
当前实现是：备份目录数量 > 2 就删除最旧的。如果有多余的非周日备份（如周三的临时备份），也会计入并可能被清理。确保只创建周日 2 点的备份目录，避免混乱。
