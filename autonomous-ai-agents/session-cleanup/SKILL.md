# 会话文件自动清理机制

## 功能概述

1. **定期清理**：每3天自动清理旧的会话文件
2. **保护新会话**：保留最近7天的会话文件
3. **汇报结果**：向主上汇报清理结果
4. **严格保护**：不删除任何系统配置和重要数据

## 清理规则

### 可清理：旧会话文件

| 条件 | 说明 |
|------|------|
| 超过7天 | 删除超过7天的会话文件 |
| 会话文件大小 | 清理大文件（>10MB）的旧会话 |

### 严格保护（禁止删除）

| 文件/目录 | 说明 |
|-----------|------|
| 最近7天 | 任何会话文件不删 |
| config.yaml | 配置文件 |
| .env | API密钥 |
| skills/ | 技能文件 |
| cron/ | 定时任务 |
| profiles/ | Profile配置 |
| backups/ | 备份文件 |

## 清理条件

- 删除超过7天的会话文件
- 单个会话文件超过10MB的旧会话
- 清理前向主上汇报

## 执行流程

```
每3天4点触发
         │
         ▼
┌─────────────────────────┐
│ 扫描会话目录            │
│ ~/.hermes/sessions/    │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 识别可清理的会话        │
│ (超过7天 或 大文件)     │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 计算清理大小            │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 执行清理                │
│ (需确认后执行)          │
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│ 向主上汇报结果          │
└─────────────────────────┘
```

## 汇报模板

### 正常情况（无需清理）

```
【会话文件清理报告】
⏰ 时间: YYYY-MM-DD 04:00

📊 会话文件状态:
   - 总文件数: 156个
   - 总大小: 1.2GB
   - 最近7天: 45个 (800MB)
   - 超过7天: 111个 (400MB)

✅ 无需清理
   - 超过7天的文件: 400MB < 阈值
   - 保留所有会话以便追溯

【清理完成】
```

### 已清理

```
【会话文件清理报告】
⏰ 时间: YYYY-MM-DD 04:00

📊 清理前状态:
   - 总文件数: 256个
   - 总大小: 2.1GB

🔄 已清理:
   - 删除会话数: 89个
   - 释放空间: 850MB
   - 最旧删除: 2026-03-15

📊 清理后状态:
   - 剩余文件: 167个
   - 剩余大小: 1.25GB
   - 最近7天: 52个

【清理完成】
```

## 手动执行

加载 skill 后可以直接说：
- "清理会话文件"
- "执行会话清理"

## 注意事项

1. 只删除超过7天的会话文件
2. 保护最近7天的会话以便问题追溯
3. 清理前会计算释放空间并汇报
4. 不会影响当前正在使用的会话
