# 中国医疗/医院信息调研模式

## 核心问题

国内医疗信息网站（好大夫在线等）有严格 Bot 检测，直接 curl/wget 抓取返回空内容。需通过搜索引擎中转获取结构化信息。

## 工作流

```
用户请求：广州肛肠科医院
         │
         ▼
┌──────────────────────────────────────┐
│ Step 1: 百度搜索（不用直接抓取源站）  │
│ 搜索词: 广州 肛肠科 医院 排行榜       │
└──────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ Step 2: 识别搜索结果中的可靠来源       │
│ ✓ 百度健康（AI生成，经专家审阅）       │
│ ✓ 好大夫在线片段（百度的内容引用）      │
│ ✗ 私立医院广告（东大肛肠等，付费排名）  │
└──────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ Step 3: 从 snippet 中提取结构化信息    │
│ - 医院名称 + 级别（三甲/专科）        │
│ - 地址                               │
│ - 科室特色                           │
│ - 权威专家                           │
└──────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ Step 4: 交叉验证（同一医院多来源确认） │
│ - 百度健康 + 搜索片段 + 地图信息       │
└──────────────────────────────────────┘
```

### 不可信的来源特征

- 搜索结果标注"广告"
- 私立肛肠专科医院（东大、民安、附大中医院等）
- 无三甲认证标记

## 工具选择

| 工具 | 适用场景 | 备注 |
|------|----------|------|
| `browser_navigate` | 直接访问目标站点 | 对好大夫等有 bot 保护的站点返回首页 |
| Bing 搜索 | 国际信息 | 效果一般 |
| 百度搜索 | 国内信息 | 提供结构化摘要和百度健康内容 |
| `browser_vision` | 从截图提取信息 | 可处理动态加载内容 |

## 搜索词模板

```
# 医院排名类
广州 + 肛肠科 + 医院 + 排行榜
广州 + 肛肠科 + 三甲医院 + 完整列表

# 科室专项类
广州 + 胃肠肛门医院 + 推荐
广州 + 痔疮 + 哪个医院比较好

# 验证类
site:haodf.com + 广州 + 肛肠科 + 医院
```

## 输出格式建议

调研结果按以下结构组织：

```
🏆 公立三甲医院（推荐优先）
   ① 医院名
      📍 地址
      🏅 地位/荣誉
      👨‍⚕️ 权威专家
      🔬 特色

🏥 其他公立综合医院

⚠️ 私立医院（注意辨别）
   - 广告多，信息需核实

💡 就诊建议
   - 常见病 → 某医院
   - 复杂病 → 某医院
   - 肿瘤 → 某医院
```
