# 文档填写流水线 Skill

## 工作原理

```
投喂源数据 → 分析模板字段 → 建立映射 → 自动填入 → 自查核对 → 输出文件 → 发送主上
```

## 支持的模板格式
- `.docx` — Word 文档（推荐，结构最完整）
- `.txt` — 纯文本模板（占位符如 {{姓名}}、{电话}、[邮箱]）
- `.md` — Markdown 模板
- `.csv` — 表格型模板（每行一条记录，填充多份）
- `.json` — JSON 结构模板

## 支持的源数据格式
- **结构化**：JSON、CSV、Excel (.xlsx)
- **半结构化**：Markdown 表格、TOML
- **非结构化**：纯文本段落、微信/邮件复制粘贴内容

> 无论主上扔过来的是什么格式，马丞相都能解析并填入。

## 使用方式

### 标准流程

**Step 1 — 告诉马丞相你的需求**
```
"加载 doc-filler 技能，我要填一份[模板类型]模板"
```

**Step 2 — 投喂两个东西给马丞相**
1. **模板文件** — 发送文件或路径
2. **源数据** — 发送文件、或直接粘贴文字内容

**Step 3 — 马丞相自动完成**
- 解析模板找出所有待填字段
- 解析源数据提取可用信息
- 建立字段映射（自动 + 主上确认）
- 填入内容
- 自查：检查必填项、格式校验、占位符残留

**Step 4 — 交付**
- 输出完成的文件到本地
- 自动发送到主上 Telegram（如需）
- 如有疑义字段，列出供主上确认

## 自查规则

马丞相会在发送前检查以下项目：

| 检查项 | 说明 |
|---|---|
| 占位符残留 | 是否还有 `{{}}` / `{}` / `[]` 未被替换 |
| 必填字段 | 是否所有必填项都已填写 |
| 格式校验 | 日期格式、手机号、邮箱、URL 等 |
| 字数/长度 | 是否有字段超过合理长度 |
| 敏感信息 | 身份证、银行卡等是否需要脱敏提醒 |
| 逻辑一致性 | 两次出现的同一字段是否一致 |

马丞相批量处理，每条记录生成一份文件，压缩包发送到主上。

## 技术实现细节

**使用的工具：**
- `read_file` — 读取模板
- `patch` — 替换占位符
- `python-docx` — Word 文档处理
- `textwrap` / `re` — 文本解析
- `send_message` — 发送文件给主上

**处理流程：**
1. 识别模板类型（docx/txt/md/csv/json）
2. 提取所有占位符（正则：`\{\{.*?\}\}`、`\{.*?\}`、`\[.*?\]`）
3. 去重并列出所有待填字段
4. 从源数据中提取匹配内容
5. 模糊匹配字段名自动建立映射（如"姓名"↔"name"）
6. 无法匹配的字段列出让主上补充
7. 执行替换，生成新文件
8. 自查通过后发送

## 常见问题

**Q：源数据和模板字段名不一样怎么办？**
A：马丞相会自动做语义匹配（如"姓名"匹配"name"/"全名"/"full_name"），匹配不上的列出让主上确认。

**Q：模板里有表格怎么填？**
A：支持。Word 表格中的 `{{字段名}}` 会被识别并填入对应单元格。

**Q：填写后想修改某几个字段怎么办？**
A：直接告诉马丞相"把姓名改成XXX，其他不变"，马丞相定位并更新后重新输出。

**Q：源数据是截图或PDF扫描件怎么办？**
A：马丞相会先用 OCR 提取文字（通过 ocr-and-documents 技能），再走正常填写流程。
